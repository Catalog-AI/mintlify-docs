The Catalog API provides endpoints for product search, discovery, and commerce intelligence. All endpoints return JSON responses.

## Base URL

```
https://api.getcatalog.ai
```

## Authentication

All requests require an API key in the `x-api-key` header. See [Authentication](/authentication) for details.

```bash
curl -H "x-api-key: YOUR_API_KEY" https://api.getcatalog.ai/v1/search
```

## Response format

All successful responses follow this structure:

```json
{
  "products": [...],  // or "product", "vendors", etc.
  "meta": {
    "totalItems": 100,
    "totalPages": 10,
    "currentPage": 1,
    "pageSize": 10
  }
}
```

## Error handling

Errors return appropriate HTTP status codes with details:

```json
{
  "error": {
    "code": "INVALID_PARAMETER",
    "message": "The 'query' parameter must be a string",
    "details": {
      "parameter": "query",
      "expected": "string",
      "received": "number"
    }
  }
}
```

| Status | Description |
|--------|-------------|
| `400` | Bad request - invalid parameters |
| `401` | Unauthorized - invalid API key |
| `429` | Rate limited - too many requests |
| `500` | Server error |

See [Error Codes](/resources/error-codes) for the complete reference.

## Rate limits

Standard accounts: 60 requests/minute, 10,000 requests/day.

See [Rate Limits](/resources/rate-limits) for details and how to handle `429` responses.

## Pagination

Most list endpoints support pagination:

```json
{
  "page": 1,
  "limit": 10
}
```

<Note>
The `totalItems` count is capped at 50 for performance. See [Pagination Guide](/guides/pagination) for implementation patterns.
</Note>
