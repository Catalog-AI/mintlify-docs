The Catalog API provides endpoints for product search, discovery, and commerce intelligence. All endpoints return JSON responses.

## Base URL

```
https://api.getcatalog.ai
```

## Authentication

All requests require an API key in the `x-api-key` header. See [Authentication](/v1/authentication) for details.

## Endpoints

### Discover

| Endpoint | Method | Description |
|----------|--------|-------------|
| [`/v1/vendors`](/v1/api-reference/endpoints/discover/get-vendors) | GET | List all vendors with product counts |
| [`/v1/collections`](/v1/api-reference/endpoints/discover/get-collections) | POST | List collections for a vendor |
| [`/v1/listings`](/v1/api-reference/endpoints/discover/get-products) | POST | List products from a vendor or collection |

### Extract

| Endpoint | Method | Description |
|----------|--------|-------------|
| [`/v1/products`](/v1/api-reference/endpoints/extract/retrieve-products) | POST | Start async batch processing of product URLs (1-1000 URLs) |
| [`/v1/products/{execution_id}`](/v1/api-reference/endpoints/extract/get-execution-status) | GET | Check status and retrieve results for a batch processing job |

### Search

| Endpoint | Method | Description |
|----------|--------|-------------|
| [`/v1/agentic-search`](/v1/api-reference/endpoints/search/agentic-search) | POST | AI-powered search with customer profile personalization |
| [`/v1/agentic-search-mini`](/v1/api-reference/endpoints/search/agentic-search-mini) | POST | Fast AI search returning up to 10 products |

### Affiliate

| Endpoint | Method | Description |
|----------|--------|-------------|
| [`/v1/affiliate`](/v1/api-reference/endpoints/affiliate/generate-links) | POST | Generate affiliate tracking links |

### Usage

| Endpoint | Method | Description |
|----------|--------|-------------|
| [`/v1/usage`](/v1/api-reference/endpoints/usage/get-usage) | GET | Get API usage statistics |

## Response format

Response structures vary by endpoint:

**Listings endpoints** (vendors, collections, listings) return:
```json
{
  "data": [...],
  "pagination": {
    "page": 1,
    "page_size": 10,
    "total_items": 100,
    "total_pages": 10,
    "has_next": true,
    "has_prev": false
  },
  "meta": {}
}
```

**Search endpoints** (agentic-search, agentic-search-mini) return:
```json
{
  "products": [...],
  "meta": {
    "totalItems": 100,
    "urls": [...]
  }
}
```

**Product execution status** returns:
```json
{
  "status": "completed",
  "results": {
    "products": [...],
    "meta": {
      "total_requested": 100,
      "total_successful": 98,
      "total_failed": 2
    }
  },
  "pagination": {
    "page": 1,
    "limit": 50,
    "total_items": 100,
    "total_pages": 2,
    "has_next": true
  }
}
```

## Error handling

Errors return appropriate HTTP status codes with structured error details:

**Standard error responses** (400, 401, 402, 403, 404, 409, 429, 500):
```json
{
  "error": {
    "code": "ERROR_CODE",
    "message": "Human-readable error description",
    "request_id": "req_..."
  }
}
```

All error responses include a `Request-ID` header that matches the `request_id` in the response body.

See [Error Codes](/resources/error-codes) for the complete reference.

## Rate limits

Default policy: 50 requests per 4-second sliding window, counted per IP with a global fallback.

See [Rate Limits](/resources/rate-limits) for headers, what `429` means, and pacing guidance.

## Pagination

All paginated responses include a `pagination` object in the response with:
- `page`: Current page number
- `page_size` or `limit`: Items per page (depends on endpoint)
- `total_items`: Total number of items
- `total_pages`: Total number of pages
- `has_next`: Whether there is a next page
- `has_prev`: Whether there is a previous page

<Note>
The `total_items` count may be capped for performance depending on the endpoint. See [Pagination Guide](/v1/guides/pagination) for implementation patterns.
</Note>
