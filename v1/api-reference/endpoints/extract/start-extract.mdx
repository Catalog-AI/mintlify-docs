---
title: "Start Product Extraction"
api: "POST https://api.getcatalog.ai/v1/extract"
description: "Start asynchronous product extraction for a merchant domain"
---

Start asynchronous product extraction for a merchant domain. This endpoint extracts products from a domain's product listings and returns immediately with an `execution_id` that you can use to check status and retrieve results.

<Note>
**Crawl Requirement:** Product extraction can only be performed on websites whose products are currently being crawled or have already been crawled. Before starting an extraction, you must first run a [crawl](/v1/api-reference/endpoints/crawl/start-crawl) to discover and index the domain's products. The extraction will use the product listings discovered during the crawl process.

**Async Processing:** This endpoint starts processing asynchronously and returns an `execution_id` immediately. Use the [Get Execution Status](/v1/api-reference/endpoints/extract/get-execution-status) endpoint to check progress and retrieve results when processing completes.
</Note>

<Tip>
**Managing Executions:** 
- **View All Executions:** You can view all your extraction executions using the [List Extract Executions](/v1/api-reference/endpoints/extract/list-executions) endpoint to see all processing jobs, their statuses, and when they were created.
- **Cancel Running Executions:** If you need to stop a running execution, use the [Cancel Extract Execution](/v1/api-reference/endpoints/extract/cancel-execution) endpoint.
- **Waiting for Crawl:** You can optionally provide a `crawl_id` to wait for a crawl execution to complete before starting extraction. This is useful when you want to ensure products are discovered before extraction begins.
</Tip>

## Request

<ParamField header="x-api-key" type="string" required>
  Your API key for authentication
</ParamField>

### Request Body

<ParamField body="domain" type="string" required>
  The domain or URL to extract products from (e.g., "example.com" or "https://example.com")
  
  **Examples:**
  - `"nike.com"`
  - `"https://www.adidas.com"`
  - `"example-store.com"`
 
</ParamField>

<ParamField body="max_products" type="number" default="0">
  Maximum number of products to process. Set to `0` for no limit (process all available products).
  
  **Note:** When set to `0`, all products found in the domain's listings will be processed.
</ParamField>

<ParamField body="enable_enrichment" type="boolean" default="true">
  Whether to enable AI enrichment for products. When enabled, products are enhanced with additional attributes and categorization.
</ParamField>

<ParamField body="country_code" type="string" default="us">
  ISO 2-letter country code for localization (e.g., "us", "ca", "gb", "de")
  
  **Supported Country Codes:**
  nl, ca, si, co, by, ee, no, br, us, de, es, vn, il, th, gb, ph, kg, in, ru, fr, hk, ua, jp, at, mm, my, pl, au, nz, ro, tw, mx, id, dk, ng, ch, hu, sg, sa, ae, cn, ar, cl, it, tr, lv, gh, sk, gr, eg, lu, bg, se, lt, lk, kz, hr, bd, kr, cz, fi, ie, be, pt, za
</ParamField>

<ParamField body="enable_reviews" type="boolean" default="true">
  Whether to enable product reviews processing
</ParamField>

<ParamField body="enable_image_tags" type="boolean" default="true">
  Whether to enable AI-generated image tags and descriptions
</ParamField>

<ParamField body="crawl_id" type="string" optional>
  Optional crawl execution ID to wait for before starting extraction. If provided, extraction will be queued and will start automatically when the crawl execution completes.
  
  **Use Case:** Use this when you want to ensure products are discovered via crawl before extraction begins. The extraction will remain in `pending` status until the crawl completes.
  
  **Format:** `crawl-{vendor}-{uuid}`
</ParamField>

## Response

<ResponseField name="execution_id" type="string">
  Unique execution identifier for this extraction job. Use this ID with the [Get Execution Status](/v1/api-reference/endpoints/extract/get-execution-status) endpoint to check progress and retrieve results.
  
  **Format:** `extract-{vendor}-{uuid}`
</ResponseField>

<ResponseField name="status" type="string">
  Initial execution status. Will be `"pending"` when starting immediately, or `"pending"` with `waiting_for` in meta when waiting for a crawl to complete.
</ResponseField>

<ResponseField name="meta" type="object">
  Execution metadata
  
  <Expandable title="Meta Properties">
    <ResponseField name="credits_used" type="number">
      Number of credits charged for this extraction
    </ResponseField>
    
    <ResponseField name="listings_count" type="number">
      Number of product listings found for this domain
    </ResponseField>
    
    <ResponseField name="products_to_process" type="number">
      Number of products that will be processed
    </ResponseField>
    
    <ResponseField name="waiting_for" type="string" optional>
      Crawl execution ID that this extraction is waiting for (only present when `crawl_id` was provided)
    </ResponseField>
  </Expandable>
</ResponseField>

## Workflow

1. **Crawl the Domain (if not already done):** First, use the [Start Crawl](/v1/api-reference/endpoints/crawl/start-crawl) endpoint to discover products from the domain. Wait for the crawl to complete or use the `crawl_id` parameter in step 2.
2. **Start Extraction:** Send a POST request with your domain to start async extraction. If the domain hasn't been crawled yet, provide the `crawl_id` from step 1 to wait for crawl completion.
3. **Get Execution ID:** Receive an `execution_id` immediately in the response
4. **Check Status:** Poll the [Get Execution Status](/v1/api-reference/endpoints/extract/get-execution-status) endpoint using the `execution_id`
5. **Retrieve Results:** When `status` is `"completed"`, results are available with pagination support
